PushButton Engine Manual - Getting Started

The first thing you will want to do is compile PBEngine. First, use the project manager, which will generate project files for a variety of common Flash development tools. Second, using your favorite supported Flash development tool, compile. Third, run your game (or a demo)!

Getting Started Even Quicker

You may want to use the engine without building it yourself. This is easily done. Once you have downloaded our precompiled SWCs, you can add them to your favorite IDE's classpath and start using the engine and components.

Step 1 - Running the Project Manager

The PushButton Project Manager is an AIR application that is bundled with the SDK. It can automatically add projects to your build environment, package projects for distribution, and create build files (currently Ant build scripts and Flex Builder projects).

To get the SDK ready for use, double click on the ProjectManager.air file included in the root of the SDK to install the Project Manager. You may need to get the Adobe AIR runtime from http://www.adobe.com/products/air/ first.

The Project Manager is associated with .pbesln files. Once it has been installed, double click on PBE.pbesln in the root of the SDK. This will launch the Project Manager and let it know where you have installed the SDK.

The first time you start the Manager it will ask for the location of the Flex SDK and the path for Eclipse/Flex Builder (if you use it). The Flex SDK path should be set to the root of the Flex SDK which has folders called ant, asdoc, bin, and several others. Provide the paths and hit save to continue.

Now, hit the "Generate Build Files..." option in the left hand column. The Project Manager will regenerate all the build files for the SDK.

At this point you can close the Project Manager and move on to step 2.

Step 2 - Compile Using...

Flex Builder

There is a batch file in Build/FlexBuilder that will launch Flex Builder with its workspace set to the Build/FlexBuilder folder. If the batch file won't run, all you need to do is open Flex Builder, go to File -> Choose Workspace -> Other... and choose Build/FlexBuilder when it asks. Hit OK and you will get a nice new workspace with a few key settings already prep'ed for you.

You may want to disable automatic builds (under Project -> Build Automatically), as they tend to slow Flex Builder down dramatically.

Once this is done, right click on the project explorer, choose Import..., then General -> Existing Projects Into Workspace.. and hit Next. Select Build/FlexBuilder as the root directory. You may have to click out of that text box for the list of projects below to update. Once that is done, make sure all the projects are selected and click Finish to import them.

The workspace should refresh, and everything should start compiling automatically.

Now you can build, browse projects, and so forth.

Ant and the Free Flex SDK

To do command line builds with the free Flex SDK and Ant, just go to the Build/FlexAnt and run the batch file contained within. You will need to have Ant installed and in your path to do this.

To get Ant, go to ant.apache.org and get the appropriate version. You may want to use an installer like WinAnt if you're on Windows, as Ant will need to be in your path.

You can build specific projects by running Ant yourself from the Build/FlexSDK folder via the command line. Do this by typing ant projectname.

FlashDevelop and Other IDEs

Most other IDEs do not have support for multiple projects, so using them with the PushButton Engine SDK can be difficult, since it uses many projects to keep things modular. However, editing any individual project is easy - create the project, point the source path of your project to the Source/ folder of the project, and include the Library/ folder in your classpath.

SDK Layout

Now that you are able to build, let's talk about how files in the SDK are organized.

  • Build/ - Contains project and build files for various development tools generated by the project manager. There is a subdirectory for each support development tool.
    • FlexBuilder/ - Contains FlexBuilder related build files.
    • FlexSDK/ - Contains Ant build files to build using the free Flex SDK.
  • Documentation/ - Contains this manual, which covers the core engine released by PushButton Labs. Third-party components' documentation is in that component's project folder (see below).
  • Libraries/ - Compiled SWCs from each library project are stored here. This is in the classpath for every project so they can access components.
  • Projects/ - One or more projects are stored in here. The project generator scans for directories with .pbeproj files when it is generating build files. Note they can be nested as deep as you like to keep things organized, and relative location doesn't matter.
    • Demos/ - Contains projects that demonstrate component libraries.
    • Engine/ - Contains projects that implement core engine features.
  • Templates/ - Stores template projects for the Project Manager's use. Templates are stored as ZIPs and their format is discussed later.

Project Layout

Each project contained in the Projects folder has the same files structure. This lets the project manager automatically generate build files, and keeps development very modular so it's easy to share components with other developers.

Projects cannot be nested within one another. The name of the project is the name of its folder. At its root, every project contains a .pbeproj file named after the project, for instance the PBEngine project contains PBEngine.pbeproj. The .pbeproj is a small XML file that describes the project, what it is, who made it, and what its dependencies are.

Projects have a type, specified in their .pbeproj file. The type can be one of:

  • Library - indicating that the project produces a SWC, named after the project. So for the RenderingTwoD project, RenderingTwoD.swc.
  • WebApplication - indicating that the project should build a SWF, in that project's Output/ subdirectory. The Source/ folder should contain a .mxml file named after the application, which is what will be compiled to produce the SWF. For the PBEngineDemo, Source/PBEngineDemo.mxml is compiled to produce Output/PBEngineDemo.swf.

The type influences the directories that are used. For a library, the structure looks like this:

  • LibraryName.pbeproj - The project description file for this library. Note it will be named after the library.
  • Documentation/ - Contains an index.html that contains or links to documentation for that library.
  • Source/ - Source files for the library. Every file in this folder or any subdirectory will be included in the library.

The results of building the library are stored as a SWC in the root Libraries folder in the SDK.

For an application, the structure is a little more complex:

  • Assets/ - The assets used by this project.
    • Images/ - Image files (jpg, png, gif, etc).
    • Levels/ - Level XML files.
    • Sounds/ - Sound files (mp3).
  • Output/ - The SWF output from the project.
  • Source/ - Source code for the project.
